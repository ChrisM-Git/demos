Get Your Credentials
CRITICAL: Save your Client Secret immediately after creation — you can never view it again after leaving the screen.
1

Check Permissions in Strata Cloud Manager

Log into https://console.paloaltonetworks.com/ and navigate to:

Common Services → Identity & Access → Access Management

You need one of:

  • Superuser for all apps and services
  • Custom role with AI Model Security access
2

Create Custom Role (if needed)

Go to Roles → Custom Roles → Create Role and enable these permissions:

  • ai_ms_pypi_auth
  • ai_ms.scans
  • ai_ms.security_groups
3

Create Service Account

Go to Service Accounts → Create Service Account, assign your role, then click Create. Copy the Client ID, Client Secret, and note your TSG ID.

4

Set Environment Variables

Mac / Linux
export MODEL_SECURITY_CLIENT_ID="your-client-id"
export MODEL_SECURITY_CLIENT_SECRET="your-client-secret"
export TSG_ID="your-tsg-id"
export MODEL_SECURITY_API_ENDPOINT="https://api.sase.paloaltonetworks.com/aims"
Windows PowerShell
$env:MODEL_SECURITY_CLIENT_ID="your-client-id"
$env:MODEL_SECURITY_CLIENT_SECRET="your-client-secret"
$env:TSG_ID="your-tsg-id"
$env:MODEL_SECURITY_API_ENDPOINT="https://api.sase.paloaltonetworks.com/aims"
To make credentials permanent on Mac/Linux, add the export lines to your ~/.bashrc or ~/.zshrc file.
Installation
The model-security-client is NOT on the public internet. It requires special authentication to download via a PyPI URL script.
1

Create the PyPI Authentication Script

Create a file called get_pypi_url.sh with this content:

get_pypi_url.sh
#!/bin/bash
set -euo pipefail

API_ENDPOINT="${MODEL_SECURITY_API_ENDPOINT:-https://api.sase.paloaltonetworks.com/aims}"
TOKEN_ENDPOINT="https://auth.apps.paloaltonetworks.com/oauth2/access_token"

TOKEN=$(curl -sf -X POST "$TOKEN_ENDPOINT" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -u "$MODEL_SECURITY_CLIENT_ID:$MODEL_SECURITY_CLIENT_SECRET" \
  -d "grant_type=client_credentials&scope=tsg_id:$TSG_ID" | \
  grep -o '"access_token":"[^"]*"' | cut -d'"' -f4)

curl -sf "$API_ENDPOINT/mgmt/v1/pypi/authenticate" \
  -H "Authorization: Bearer $TOKEN" | \
  grep -o '"url":"[^"]*"' | cut -d'"' -f4
Make Executable
chmod +x get_pypi_url.sh
2

Install the Model Security Client

Mac / Linux
pip install pan-aisecurity flask
pip install model-security-client --extra-index-url $(./get_pypi_url.sh)
3

Verify Installation

Terminal
model-security-client --version
# Expected output: 1.0.0 or similar
Running the Web Scanner
1

Start the Web Interface

Terminal
python3 modelscan_app.py

# Expected output:

2

Open & Configure

Navigate to http://localhost:5001 in your browser, enter your CLIENT_ID, CLIENT_SECRET, and TSG_ID, then click Save Configuration.

3

Choose Your Scan Type

  • 🤗 HuggingFace — enter a HuggingFace model URL
  • 📁 Local File — select a model file from your computer
  • ☁️ S3 / Cloud — enter an S3 URI (s3://bucket/model)
Large models can take 10–15 minutes to scan. Keep the terminal running.
CLI Reference
Scan HuggingFace Model
model-security --base-url "https://api.sase.paloaltonetworks.com/aims" scan \
  --security-group-uuid "e3adbe4f-4481-4bd9-9c7c-33b5123d40bf" \
  --model-uri "https://huggingface.co/microsoft/DialoGPT-medium"
Scan Local Model
model-security --base-url "https://api.sase.paloaltonetworks.com/aims" scan \
  --security-group-uuid "d9778b1c-cf1b-493d-a1ca-ef2a99d2bae4" \
  --model-path "/path/to/your/model"
Scan S3 Model
model-security --base-url "https://api.sase.paloaltonetworks.com/aims" scan \
  --security-group-uuid "a165a2a2-e864-4426-883d-fe623646a014" \
  --model-uri "s3://your-bucket/your-model"
Set export MODEL_SECURITY_BASE_URL="https://api.sase.paloaltonetworks.com/aims" to skip the --base-url flag on every command.

Troubleshooting

"Command not found: model-security-client"

Scanner isn't installed or not in PATH. Reinstall:

Fix
pip install model-security-client --extra-index-url $(./get_pypi_url.sh)
pip list | grep model-security # verify

"Authentication failed"

Environment variables not set correctly. Check and reset:

Fix
echo $MODEL_SECURITY_CLIENT_ID  # should not be empty
echo $TSG_ID
export MODEL_SECURITY_CLIENT_ID="..."  # reset if empty

"Cannot connect to localhost:5001"

The web app isn't running. Start it and check the port:

Fix
python3 modelscan_app.py
lsof -i :5001  # Mac/Linux — check port
netstat -ano | findstr :5001  # Windows

"Virtual environment deactivated"

Fix
source venv/bin/activate   # Mac/Linux
venv\Scripts\activate      # Windows
