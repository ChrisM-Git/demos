Prerequisites

Active License

Prisma AIRS AI Red Teaming license required

IAM Permissions

Superuser or Read-only role with Red Teaming access

Deployment Profile

Active deployment profile configured in SCM

If a deployment profile exists but IAM permissions for AI Red Teaming are missing, authentication errors will occur. Both are required.
Verify IAM Permissions
1

Navigate to Strata Cloud Manager

Go to https://console.paloaltonetworks.com/

Common Services → Identity & Access → Access Management

2

Confirm Required Role

Select your tenant and verify one of the following is assigned:

  • Superuser (all applications and services)
  • Read-only role with explicit AI Red Teaming access
Network Broker Setup

Required Network Endpoints

Your environment must allow outbound/bidirectional access to:

  • api.sase.paloaltonetworks.com
  • auth.apps.paloaltonetworks.com
  • registry.ai-red-teaming.paloaltonetworks.com

Target domains must be DNS resolvable.

1

Create Docker Pull Secret

kubectl
kubectl create secret docker-registry airs-pull-secret \
  --docker-server=registry.ai-red-teaming.paloaltonetworks.com \
  --docker-username=<USERNAME> \
  --docker-password=<PASSWORD>
The Docker pull secret is shared across the entire tenant and only needs to be created once.
2

Authenticate to Helm Registry

helm
helm registry login registry.ai-red-teaming.paloaltonetworks.com \
  --username=<USERNAME> \
  --password=<PASSWORD>
3

Install Network Client via Helm

helm install
helm install panw-network-client \
  oci://registry.ai-red-teaming.paloaltonetworks.com/pairs-redteam-prd-fckx/red-teaming-onprem/charts/panw-network-client:1.0.2 \
  --set config.clientId=<CLIENT_ID> \
  --set config.clientSecret=<CLIENT_SECRET> \
  --set config.channelId=fdec449f-3116-4352-a4b4-fbb421f3aa8a

Configure Red Teaming Targets

Target Types

  • Model — A standalone AI model endpoint
  • Application — An AI-powered application (RAG, chatbot)
  • Agent — Autonomous AI that invokes tools, APIs, or external systems
Select Agent only if the target independently invokes tools or APIs, determines action sequences, or interacts with external systems.
1

Create a New Target

In SCM navigate to AI Security → AI Red Teaming → Targets → + New Target

Tip: Use Save for Later at any point — saved targets appear with Draft status.

2

For OpenAI / HuggingFace Targets

Provide:

  • API Key
  • API Endpoint (e.g. https://api.openai.com/v1/chat/completions)
  • Model Name (e.g. gpt-3.5-turbo)
3

For Custom REST API Targets

Provide the endpoint URL, request format, and expected response structure. Example:

Request Example
curl "http://your-endpoint/chat" \
  -H "Content-Type: application/json" \
  --data '{"message": "{INPUT}"}'
Expected Response JSON
{
  "response": "{RESPONSE}",
  "security_violation": null
}
{INPUT} = prompt sent by Red Teaming  |  {RESPONSE} = target's reply
4

Configure Target Background

Provide context to improve scan accuracy — mandatory for Applications and Agents:

  • Industry
  • Use Case (e.g. customer support, code generation)
  • Optional: Competitors, PDF documentation uploads

Use Agentic Profiling to auto-populate fields via AI-driven endpoint analysis.

Security & Safety Drift Detection

What Is Drift?

Over time, models can become more vulnerable to attacks or less compliant with safety policies without active monitoring. Input patterns evolve, new attack vectors emerge, and previously effective guardrails may weaken.

Scenario Example

TimeframeObservation
Month 1 (Baseline)Model blocks 95% of prompt injection, 98% safety compliance
Month 3New jailbreak techniques discovered, edge cases not in original testing
ResultModel begins allowing prohibited outputs — drift detected

Measurable Drift Indicators

  • Increased success rate of prompt injection attacks
  • Higher bypass rates for content filtering
  • Degraded detection of adversarial inputs
  • Declining compliance with safety policies across protected topics
  • Weakening of refusal behaviors for prohibited requests
Without the tool: Degradation remains unnoticed until incidents occur. With the tool: Drift is detected early through continuous automated testing, enabling corrective action before exploitation.
